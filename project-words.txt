# quickchr project dictionary (cspell)
#
# Canonical home for project vocabulary — QEMU, RouterOS, networking, and
# tikoci domain terms absent from the default en dictionary. Referenced from
# cspell.json, so both the editor extension and CLI `cspell lint` (run via
# `bun run lint:cspell` / `bun run check`) read it. This replaces the old
# `.vscode/settings.json` `cSpell.words` list, which the CLI could not see.
#
# cSpell matches case-insensitively, so the lowercase form covers all casings
# (e.g. "respawn" -> "Respawning", "qemu" -> "QEMU"). Keep sorted; add a word
# rather than tolerate noise. The editor "Add to dictionary" quick-fix appends
# here automatically (dictionaryDefinitions addWords: true). Locale variants
# (British spellings), doc placeholders (my*), and one-off test sentinels live
# in cspell.json `ignoreWords`, not here.
